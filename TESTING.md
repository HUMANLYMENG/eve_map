# 测试指南

## 快速开始

### 1. 确保本地服务器在运行

```bash
cd /Users/ke/Documents/eve-regional-map
python3 -m http.server 8080
```

### 2. 访问测试页面

打开浏览器访问以下 URL：

| 页面 | URL | 用途 |
|------|-----|------|
| 模块验证 | http://localhost:8080/verify.html | 快速检查所有模块是否能加载 |
| 功能测试 | http://localhost:8080/test.html | 完整功能测试（带测试面板） |
| 原版对比 | http://localhost:8080/index.html | 原始版本 |

### 3. 验证步骤

#### 步骤 1: 模块加载验证
访问 http://localhost:8080/verify.html

预期结果：
```
✓ core/index.js 加载成功
✓ features/index.js 加载成功
✓ data/index.js 加载成功
✓ renderer/index.js 加载成功
✓ interaction/index.js 加载成功
✓ ui/toast.js 加载成功
✓ app.js 加载成功
```

#### 步骤 2: 功能测试
访问 http://localhost:8080/test.html

点击测试面板中的按钮：
- "测试核心模块" - 验证工具函数和事件总线
- "测试数据模块" - 验证数据管理器
- "测试功能模块" - 验证虫洞和路径记录
- "运行单元测试" - 运行所有测试

#### 步骤 3: 功能对比
1. 打开原版 http://localhost:8080/index.html
2. 打开新版 http://localhost:8080/test.html
3. 对比功能是否一致：
   - 星域切换
   - 地图平移/缩放
   - 星系点击选中
   - 搜索功能
   - 路径记录

## 预期行为

### 正常情况
- 页面加载后显示默认星域（金纳泽）
- 可以选择其他星域
- 可以拖拽平移地图
- 可以滚轮缩放地图
- 点击星系显示信息并记录路径

### 可能的问题

#### 问题 1: 模块加载失败
**症状**: verify.html 显示红色错误  
**解决**: 检查文件路径是否正确，确保使用 http:// 而非 file://

#### 问题 2: 数据加载失败
**症状**: 显示 "加载星图数据..." 一直转圈  
**解决**: 检查 data/ 目录是否存在 YAML/JSONL 文件

#### 问题 3: 渲染错误
**症状**: 地图不显示或显示异常  
**解决**: 打开浏览器开发者工具查看 Console 错误

## 回滚方法

如果测试发现问题，可以回滚到旧版本：

```bash
# 恢复旧的主文件
git checkout -- js/main.js js/dataLoader.js js/mapRenderer.js js/interaction.js

# 或者手动修改 index.html，使用旧版脚本引用
```

## 调试技巧

1. **打开开发者工具**: F12 或右键-检查
2. **查看 Console**: 检查是否有错误信息
3. **网络面板**: 检查模块是否正确加载 (状态码 200)
4. **Sources 面板**: 检查源码映射是否正确

## 已知限制

1. 必须使用本地服务器（http://localhost:8080）
2. 需要现代浏览器（不支持 IE）
3. 首次加载数据需要 1-2 秒
